# Agent Rules

本文件是当前仓库的 Agent 入口规则。进入本项目后，优先遵循这里的规则；更详细的流程说明引用 `doc/` 下的规则文档。

## 基本工作规则

- 默认先阅读相关代码和文档，再开始修改。
- 修改前先说明影响范围。
- 只做和当前任务相关的最小改动。
- 不修改无关文件。
- 提交前必须运行与改动范围匹配的最小验证。
- 说明使用中文，代码、命令、文件名保持英文。
- 若工作区已有无关改动，不要回滚、格式化或提交它们。

## 计划规则

- 如果任务预计会修改超过一个文件，先遵循 `doc/PLAN.md` 输出实现计划。
- 计划需要说明要解决的问题、影响模块、3-7 个步骤、每步验证方式、易错点。
- 在用户确认计划前，不执行实际代码或文档修改。
- 用户明确要求实现已确认计划时，可以开始执行。

## 完成输出规则

任务完成后，按 `doc/EndOutput.md` 汇报：

1. 修改了哪些文件。
2. 每个文件为什么改。
3. 运行了哪些验证。
4. 还有哪些没验证。
5. 哪些地方需要人工确认。

## 仓库约束

- 当前仓库是 HarmonyOS NEXT 客户端工程。
- 后端在独立工程中实现；不要在本仓库新增后端服务代码，除非用户明确改变该边界。
- 文档、客户端代码和配置改动应保持边界清晰。
